# Code of Conduct

## Purpose

STAFF App API is intended to be a professional, welcoming and technically
serious open source project. We expect participants to help create a community
where contributors can discuss software, security, health-adjacent workflows and
architecture with care and respect.

This Code of Conduct applies to project spaces including issues, pull requests,
discussions, reviews, documentation, community chat and public representation of
the project.

## Expected Behavior

Participants are expected to:

- be respectful of different experience levels and backgrounds;
- give and receive technical feedback constructively;
- focus criticism on code, documentation, design and evidence;
- avoid sharing private, sensitive or personal information;
- disclose conflicts of interest when they affect technical decisions;
- follow project security, privacy and licensing policies;
- assume responsibility for mistakes and correct them promptly.

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, intimidation, threats or personal attacks;
- discriminatory language or conduct;
- sexualized language, imagery or attention;
- publishing private information without explicit permission;
- sharing real health, student, coach, credential or production data;
- repeated disruption of technical discussions;
- knowingly submitting malicious code, insecure guidance or unlicensed content;
- retaliation against someone who reports a concern in good faith.

## Health-Adjacent Domain Expectations

Because this project supports training and anamnese workflows, contributors must
avoid presenting software output as medical advice. Discussions about exercise,
rehabilitation, contraindications or risk scoring should be framed as software
behavior and should respect professional clinical judgment.

## Reporting

Report Code of Conduct concerns privately to the maintainers through the
project's official private contact channel once the public repository is active.

Do not open a public issue if the report includes:

- personal information;
- private messages;
- security details;
- health-related data;
- retaliation risk.

Maintainers will review reports in good faith, protect reporter privacy as far
as reasonably possible and take action proportionate to the situation.

## Enforcement

Maintainers may take one or more actions:

- private clarification or warning;
- request to edit or remove content;
- temporary restriction from project spaces;
- removal from maintainer roles;
- permanent restriction from project participation.

Severe threats, malicious activity or repeated violations may lead to immediate
restriction without prior warning.
